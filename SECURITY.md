# Security Policy

Do not report security issues through public issues or casual chat threads.

## Report Privately

- Contact: `architects@zer0pa.ai`

Include:

- affected file or surface
- reproduction steps or proof of concept
- severity and impact
- any remediation proposal

## What Counts

- committed secrets or credentials
- package or workflow issues that could enable code execution, privilege escalation, or data exfiltration
- dependency vulnerabilities in the declared package surface

## What Does Not Count

- inconclusive proof findings
- benchmark misses
- normal engineering regressions without a security impact
