# Security Policy

## Supported Scope

This policy covers the current `main` branch, tagged releases published from
it, the root Python package install surface, the optional native helper built
from the matching checkout, and the documentation and proof artifacts shipped
in this repository.

Environment-specific market-data outages, third-party account issues, and
general support questions are not treated as security vulnerabilities unless
they create a confidentiality, integrity, or availability impact in the
shipped ZPE-FT surface.

## Reporting

Report vulnerabilities privately to `architects@zer0pa.ai`. Include the
affected version or commit, a minimal reproduction path, environment details,
and the impact you observed or expect.

Do not open a public GitHub issue for an untriaged security report. If you are
unsure whether something is security-sensitive, send it privately first and we
will triage it there.

## Response Targets

- Acknowledgement within 5 business days.
- Initial triage status or follow-up questions within 10 business days.
- After triage, a remediation plan, mitigation guidance, or accepted-risk
  decision shared on the post-triage timeline appropriate to severity and
  reproducibility.
