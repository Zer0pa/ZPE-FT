# Security Policy

Do not report security issues through casual chat or public summaries.

<p>
  <img src=".github/assets/readme/section-bars/reporting-a-vulnerability.svg" alt="REPORTING A VULNERABILITY" width="100%">
</p>

Report privately to:
- `architects@zer0pa.ai`

Include:
- affected file or surface
- reproduction steps or proof of concept
- severity and impact
- any remediation proposal

<p>
  <img src=".github/assets/readme/section-bars/what-we-accept.svg" alt="WHAT WE ACCEPT" width="100%">
</p>

- committed secrets or credentials
- package or workflow paths that enable code execution, privilege escalation,
  or data exfiltration
- dependency vulnerabilities in the declared package surface
- security defects in the helper build, install, or release workflow

<p>
  <img src=".github/assets/readme/section-bars/what-we-do-not-accept.svg" alt="WHAT WE DO NOT ACCEPT" width="100%">
</p>

- inconclusive proof findings without a security impact
- benchmark misses or ordinary model-performance debate
- historical path defects already marked as historical-only unless they expose a
  new active risk

<p>
  <img src=".github/assets/readme/section-bars/response-expectations.svg" alt="RESPONSE EXPECTATIONS" width="100%">
</p>

Security reports are prioritised over normal repo questions. This document is a
routing policy only; it does not change the legal terms in [`LICENSE`](LICENSE).
