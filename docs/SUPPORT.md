`LICENSE` is the legal source of truth for licensing terms. This support guide
is a routing summary only and is not legal advice.

<p>
  <img src="../.github/assets/readme/section-bars/what-gets-a-fast-response.svg" alt="WHAT GETS A FAST RESPONSE" width="100%">
</p>
Reports with:

- a reproducible command
- a concrete file or artifact path
- a rendered-doc problem that can be located in-repo
- a specific contradiction between prose and retained evidence

<p>
  <img src="../.github/assets/readme/section-bars/what-does-not-get-a-response-here.svg" alt="WHAT DOES NOT GET A RESPONSE HERE" width="100%">
</p>
- requests to upgrade bounded evidence into broader authority without new proof
- claim disputes with no counter-evidence artifact
- questions already answered in `README.md`, `docs/FAQ.md`, or `docs/README.md`
- vague summaries with no file path, artifact path, or command

<p>
  <img src="../.github/assets/readme/section-bars/response-expectations.svg" alt="RESPONSE EXPECTATIONS" width="100%">
</p>
This is a source-available project maintained by a small team. Security reports
and evidence contradictions are prioritised over general questions. Public
response timing is best-effort rather than a formal SLA.
