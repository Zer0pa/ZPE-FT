# Support

<p>
  <img src="../.github/assets/readme/section-bars/where-to-go.svg" alt="WHERE TO GO" width="100%">
</p>

| What you need | Where to go |
|---|---|
| Package or docs bug | private repo issue with the exact file path and command; if you do not have repo access, email `architects@zer0pa.ai` with the same details |
| Evidence dispute | private repo issue with the contradicted artifact path; if you do not have repo access, email `architects@zer0pa.ai` with the artifact path and contradiction |
| Security vulnerability | `architects@zer0pa.ai` |
| Licensing or commercial question | `architects@zer0pa.ai`; [`LICENSE`](../LICENSE) is the legal source of truth |
| Contribution guidance | [`../CONTRIBUTING.md`](../CONTRIBUTING.md) |
| Release posture question | [`../RELEASING.md`](../RELEASING.md) |
| Common reader question | [`FAQ.md`](FAQ.md) |

<p>
  <img src="../.github/assets/readme/section-bars/what-gets-a-fast-response.svg" alt="WHAT GETS A FAST RESPONSE" width="100%">
</p>

- a reproducible command
- a concrete artifact path
- a rendered-doc path problem
- a precise contradiction between prose and shipped evidence

<p>
  <img src="../.github/assets/readme/section-bars/what-does-not-get-a-response-here.svg" alt="WHAT DOES NOT GET A RESPONSE HERE" width="100%">
</p>

- requests to upgrade bounded evidence into broader authority without new proof
- claims disputes with no counter-evidence
- questions already answered in the root README or FAQ
- public chat, public issue, or vague summary with no file, command, or
  artifact path

<p>
  <img src="../.github/assets/readme/section-bars/response-expectations.svg" alt="RESPONSE EXPECTATIONS" width="100%">
</p>

This repo does not promise a public SLA in documentation. Security and evidence
contradictions are treated as higher priority than general questions.
